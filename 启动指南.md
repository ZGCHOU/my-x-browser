# Camoufox 多账号管理平台 - 启动指南

## 项目简介

基于 Camoufox 的多账号指纹浏览器管理工具，支持多账号隔离、代理配置、图形化管理界面。

## 快速启动

### 1. 环境准备

确保已安装 Python 3.7+：

```bash
python3 --version
```

### 2. 激活虚拟环境

项目已包含虚拟环境，直接激活：

```bash
source venv/bin/activate
```

### 3. 安装依赖（首次运行）

如果是首次使用或依赖未安装：

```bash
pip install -r requirements.txt
```

依赖包括：
- `camoufox` - 指纹浏览器核心
- `PyQt5` - 图形界面库

### 4. 启动管理平台

```bash
python run_manager.py
```

或者直接执行：

```bash
./run_manager.py
```

## 使用说明

### 添加账号

1. 点击界面上的 **"+ 添加账号"** 按钮
2. 填写账号信息：
   - **账号名称**（必填）：用于标识不同账号
   - **代理地址**（可选）：格式如 `http://127.0.0.1:7897` 或 `socks5://127.0.0.1:1080`
   - **备注**（可选）：账号用途说明

### 启动浏览器

1. 在左侧列表选择要启动的账号
2. 在 **"启动URL"** 输入框填写目标网址（如 `https://www.google.com`）
3. 点击 **"🚀 启动浏览器"**
4. 浏览器将在独立进程中启动，配置完全隔离

### 停止浏览器

1. 选择正在运行的账号（状态显示为运行中）
2. 点击 **"🛑 停止浏览器"**

### 删除账号

1. 确保账号的浏览器已停止
2. 选择要删除的账号
3. 点击 **"🗑️ 删除账号"**
4. 注意：删除账号不会删除 `profiles/` 目录下的配置文件

## 测试脚本

项目包含几个测试脚本，可用于验证功能：

### 基础启动测试

```bash
python start.py
```

功能：
- 自动扫描并加载 `~/.cache/camoufox/addons` 下的插件
- 访问指纹检测页面验证伪装效果
- 使用本地代理 `http://127.0.0.1:7897`

### 修复并启动

```bash
python fix_and_start.py
```

功能：
- 清理无效插件目录
- 尝试带插件启动，失败则降级为纯净模式
- 适合排查插件相关问题

## 项目结构

```
camoufox/
├── manager/                    # 管理平台核心
│   ├── core/                   # 核心逻辑
│   │   ├── account_manager.py  # 账号管理
│   │   └── browser_manager.py  # 浏览器进程管理
│   ├── gui/                    # 图形界面
│   │   ├── main_window.py      # 主窗口
│   │   └── styles.py           # 样式定义
│   └── data/                   # 数据存储
│       └── accounts.json       # 账号数据（自动生成）
├── profiles/                   # 浏览器配置文件（按账号隔离）
├── venv/                       # Python 虚拟环境
├── run_manager.py              # 主启动脚本
├── start.py                    # 测试脚本
├── fix_and_start.py            # 修复启动脚本
└── requirements.txt            # 依赖列表
```

## 配置说明

### 代理配置

支持的代理格式：

```
http://127.0.0.1:7897
https://proxy.example.com:8080
socks5://127.0.0.1:1080
```

### 数据存储位置

- **账号数据**：`manager/data/accounts.json`
- **浏览器配置**：`profiles/<账号名>/`
- **插件缓存**：`~/.cache/camoufox/addons/`

### 配置隔离

每个账号拥有独立的：
- Cookie 存储
- 浏览器指纹
- 本地存储
- 缓存数据

## 常见问题

### 1. 启动失败

检查虚拟环境是否激活：
```bash
which python  # 应该指向 venv/bin/python
```

### 2. 依赖缺失

重新安装依赖：
```bash
pip install --upgrade -r requirements.txt
```

### 3. 代理连接失败

确认代理服务正在运行：
```bash
curl -x http://127.0.0.1:7897 https://www.google.com
```

### 4. 浏览器无法启动

尝试使用修复脚本：
```bash
python fix_and_start.py
```

## 退出程序

- 关闭管理平台窗口会自动停止所有正在运行的浏览器
- 或使用 `Ctrl+C` 终止进程

## 注意事项

1. 建议为不同用途的账号配置不同的代理
2. 定期备份 `manager/data/accounts.json` 和 `profiles/` 目录
3. 删除账号前确保已保存重要数据
4. 首次启动可能需要下载浏览器组件，请耐心等待

## 技术支持

如遇问题，请检查：
- Python 版本是否 >= 3.7
- 虚拟环境是否正确激活
- 依赖是否完整安装
- 代理配置是否正确
